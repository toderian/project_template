# Prototype

Use this skill to build throwaway code that answers a specific design question — a logic/state question via an interactive TUI, or a UI question via several variant pages.

Read and follow:

- `playbooks/skills/engineering/prototype.md`
- `playbooks/skills/engineering/prototype/LOGIC.md` (if the question is about state/business logic)
- `playbooks/skills/engineering/prototype/UI.md` (if the question is about visual design)

Keep this skill thin. The playbook is the shared workflow and should be updated first when the process changes.
