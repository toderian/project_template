# Edit Article

## Purpose

Edit and improve articles through multi-pass revision: structure first, then clarity, then prose tightening.

## Workflow

### Pass 1: Structure

1. Divide the article into sections based on its headings.
2. Map the information dependency graph — pieces of information can depend on other pieces. Ensure the section order respects these dependencies.
3. Identify sections that are missing, redundant, or in the wrong position.
4. Confirm the proposed structure with the user before proceeding.

### Pass 2: Clarity

For each section:

1. Identify the single main point of the section.
2. Rewrite to lead with that point, then support it.
3. Remove sentences that do not advance the section's purpose.
4. Replace jargon with plain language unless the audience expects it.
5. Ensure transitions between sections are smooth — the reader should never wonder "why am I reading this now?"

### Pass 3: Prose tightening

For each paragraph:

1. Target maximum 240 characters per paragraph — shorter is better for technical content.
2. Cut filler words: "very", "really", "basically", "in order to", "it should be noted that".
3. Prefer active voice over passive.
4. One idea per sentence. If a sentence has two ideas, split it.

### Presenting changes

- Show edits inline with the original so the user can compare.
- Explain why you made non-obvious structural changes.
- Do not rewrite voice or tone unless the user asks — preserve the author's style.

## Quality bar

- Every section has a clear purpose the reader can identify in the first sentence
- No paragraph exceeds 240 characters
- Information order respects the dependency graph
- The article can be skimmed by reading only the first sentence of each section
