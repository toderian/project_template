# Reference docs

Project documentation lives here — architecture notes, component `CONTEXT.md` files, design
decisions, runbooks, and anything else that explains *how the system works* (as opposed to *what work
is queued*, which lives in `../tasks_manager/`).

This folder is seeded empty by `/init`. Add files as the project grows; there's no required layout.
`/describe-component` writes component architecture docs here when `CONTEXT_DOCS_DIR` points at it.
