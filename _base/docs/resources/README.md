# Resources

Durable project documentation lives here: architecture notes, component `CONTEXT.md` files, decisions,
runbooks, research notes, and other material that explains how the system works.

Queued work belongs in `../tasks_manager/`. Area status belongs in `../areas/`. Frozen or obsolete
docs/resources belong in `../archive/`.

This folder is seeded empty by `/init`; add structure as the project grows.
