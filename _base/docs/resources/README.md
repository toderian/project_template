# Resources

Durable project documentation lives here: the primary domain glossary (`CONTEXT.md`), component
contexts under `<area>/components/<component-slug>/CONTEXT.md`, decisions, runbooks, research notes,
and other material that explains how the system works.

Queued work belongs in `../tasks_manager/`. Area status belongs in `../areas/`. Frozen or obsolete
docs/resources belong in `../archive/`.

This folder is seeded by `/init`; add structure as the project grows.
