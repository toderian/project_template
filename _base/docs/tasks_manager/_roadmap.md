# Roadmap

The plan of execution - what is **Urgent**, **Now**, **Next**, **Later**, and **Someday**. Task IDs
(`AUTH-001`, `T-001`) may appear in any horizon; raw inbox ideas (`I-NNN`) may appear only in
**Someday**. The horizon (placement) is a deliberate, human-set sequence; it is *not* auto-derived from
task status or priority.

Keep this file placement-only: one bullet per task ID, or one inbox ID in **Someday** only. Task files
remain authoritative for status, priority, phases, acceptance criteria, and completion state.

Soft review thresholds:

- **Urgent**: interrupting work, 0-2 items.
- **Now**: active/immediate pickup, 1-3 items.
- **Next**: committed near-term queue, 5-10 items.
- **Later**: valid committed tasks, intentionally unscheduled.
- **Someday**: weak commitment / parking lot.

Threshold pressure should be mentioned during roadmap review, but it is not a validation failure.
